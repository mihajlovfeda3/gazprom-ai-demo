# Демо-сценарий: поиск проверенных материалов под рабочую задачу

## Цель демонстрации

Показать новый pivot-сценарий `gazprom-ai-demo`: система не оценивает сотрудника и не строит карьерный skill gap, а помогает быстро найти и объяснить релевантные внутренние корпоративные материалы под рабочую задачу или цель развития.

Главный endpoint демонстрации:

```http
POST /api/material-agent
```

## Основной сценарий

### 1. Пользователь вводит задачу

Фраза для демонстрации:

> Нужно разобраться, как описывать API и интеграции в проекте

### 2. Агент выделяет темы

Агент определяет темы запроса:

- API design
- интеграции

Важно проговорить: это не оценка навыков сотрудника и не skill gap, а выделение тем из рабочей задачи.

### 3. Агент ищет по внутренней базе

Агент ищет материалы в корпоративной базе знаний:

- внутренние курсы
- НМД
- методички
- документы
- видео-транскрипты

External courses не возвращаются в `material-agent`.

### 4. Агент ранжирует материалы

Агент показывает `match_label` и процент совпадения, например:

- `92% совпадения`
- `87% совпадения`
- `82% совпадения`

Ранжирование остается deterministic и не выполняется LLM.

### 5. Агент показывает карточки материалов

В карточке материала важно показать:

- название
- тип материала
- источник
- ответственного или владельца
- актуальность
- reason

Ключевые поля UI:

- `title`
- `material_type_label`
- `source`
- `responsible_label`
- `actuality_label`
- `match_label`
- `reason`

### 6. Агент показывает quality alerts

Если видео-транскрипт или материал требует проверки владельцем, агент возвращает `quality_alerts`.

Примеры смыслов:

- материал требует проверки актуальности владельцем
- видео-транскрипт распознан, но еще не подтвержден владельцем

### 7. Qwen формирует короткое объяснение

Если локальная Ollama доступна, `answer_mode = llm`, а `llm_model = qwen3:0.6b`.

Qwen формирует короткое объяснение: почему найденные материалы подходят под задачу.

Если Ollama недоступна или ответ LLM не проходит safety/format checks, возвращается `answer_mode = template_fallback`.

### 8. Что важно проговорить на защите

Система не оценивает сотрудника.

Система не назначает обучение.

Система не строит карьерный маршрут.

Система не является источником истины.

Система ускоряет поиск и объясняет релевантность материалов.

Решение о применении материала, согласовании времени или дальнейшем обучении остается за человеком и корпоративным процессом.

## Guardrail-сценарий

### 1. Вводим нерелевантный запрос

Фраза для демонстрации:

> как дела квен

### 2. Агент возвращает guardrail

Ожидаемый результат:

- `answer_mode = guardrail`
- `recommended_materials = []`
- `sources = []`
- `quality_alerts = []`

### 3. Что важно проговорить

Для нерелевантного запроса агент не запускает поиск по базе знаний и не вызывает LLM-генерацию.

Он объясняет допустимую область: поиск корпоративных материалов, курсов, НМД, видео-транскриптов и документов под рабочую задачу или цель развития.
