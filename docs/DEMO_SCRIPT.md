# Demo Script — запуск MVP на защите

## Цель демо

Показать не production-систему, а рабочую механику будущей T&D-платформы:

```txt
пользовательский интерфейс → AI-агенты → backend → RAG-lite поиск → курсы / источники / ИПР → согласование руководителем
```

## Короткая формулировка

Мы собрали демонстрационный MVP: frontend показывает пользовательские сценарии, backend реализует RAG-lite поиск по базе знаний, расчет разрыва компетенций, подбор курсов, формирование ИПР и передачу маршрута руководителю. В production этот контур переносится на корпоративный RAG + Mistral Small / Medium внутри защищенного периметра.

---

## Сценарий демо на 2–3 минуты

### 1. Стартовый экран

Говорим:

> Сейчас покажем путь сотрудника, который хочет перейти из роли Middle системного аналитика в Senior системного аналитика. В демо мы используем JSON-базу и RAG-lite поиск, чтобы показать механику будущей платформы.

Действие:

- выбрать текущую роль: `Middle системный аналитик`
- выбрать целевую роль: `Senior системный аналитик`
- нажать кнопку формирования маршрута

---

### 2. Маршрут развития

Говорим:

> Агент маршрута сравнивает текущие компетенции сотрудника с матрицей целевой роли и считает разрыв. Это не просто подбор курсов по ключевому слову, а связка “целевая роль → недостающие навыки → релевантные курсы → ИПР”.

Показать:

- карточку сотрудника
- `skill_gap`
- recommended courses
- источники

Акцент:

- курсы имеют провайдера
- виден тип курса: внутренний / внешний
- видна цена
- видно, нужно ли согласование руководителя

---

### 3. Черновик ИПР

Говорим:

> На основе найденных курсов система формирует черновик индивидуального плана развития. Сотрудник не собирает его вручную из разных систем, а получает обоснованный маршрут с источниками.

Показать:

- `ipr_draft`
- summary
- источники RAG

---

### 4. Передача руководителю

Говорим:

> Если обучение требует времени или бюджета, маршрут передается руководителю. Руководитель видит не просто заявку на курс, а контекст: роль сотрудника, цель развития, загрузку и рекомендацию.

Действие:

- нажать “Отправить руководителю”
- открыть экран руководителя

Показать:

- manager
- employee
- workload
- recommendation
- decision_options: согласовать / скорректировать / отложить

---

### 5. Свободный вопрос агенту знаний

Говорим:

> Второй сценарий — агент корпоративного знания. Сотрудник может задать свободный вопрос, например: какие курсы помогут прокачать системный дизайн и интеграции. Агент ищет по демо-базе курсов и источников и возвращает ответ с найденными материалами.

Действие:

Ввести вопрос:

```txt
Какие внешние курсы помогут прокачать системный дизайн и интеграции?
```

Показать:

- answer
- related_courses
- sources

---

## Что важно проговорить жюри

1. Мы не заменяем существующие системы, а показываем интеллектуальный слой, который может встроиться поверх LMS, карьерного портала, ИПР и данных руководителя.
2. В демо нет настоящей LLM, чтобы не зависеть от внешних API. Ответы формируются шаблоном, а поиск работает через TF-IDF.
3. В production RAG-lite заменяется корпоративным RAG, а template generator — развернутыми внутри контура LLM.
4. Главная ценность — не “ИИ рекомендует курсы”, а управляемый сценарий: компетенции → курсы → ИПР → согласование → источники.

## Что не надо говорить

Не говорить:

- “Мы сделали полноценную платформу”.
- “Это можно сразу внедрять в production”.
- “ИИ сам принимает карьерные решения”.
- “Система заменяет карьерных консультантов”.

Лучше говорить:

> Это MVP-прототип, который демонстрирует целевую механику и пользовательский путь. В production решение должно пройти интеграцию с корпоративными системами, проверку безопасности, настройку прав доступа и подключение к проверенным источникам знаний.
